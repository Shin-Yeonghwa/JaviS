1. Ajax

  • 페이지를 전환하지 않고 서버에서 데이터를 받아와 사용자에게 보여줌
  • 현재 대부분의 웹사이트가 Ajax를 도입

2-1. 데이터 전송 형식 - CSV형식

  • 다른 형식에 비해 매우 짧고, 많은 양의 데이터를 전송할 때 활용
  • 데이터의 가독성이 떨어지는 단점 (각 데이터가 무엇을 나타내는지 알기 힘듦)
  • split() 메서드로 분해

2-2. 데이터 전송 형식 - XML 형식

  • HTML처럼 태그로 데이터를 표현
    <name>김혜선</name>
    <age>37</age>
  • 대부분의 정보 제공 사이트에서 제공하는 RSS형식도 XML 형식을 기반으로 만듦
  • 용량이 크고, 데이터의 양이 커지면 분석 속도가 떨어짐

2-3.데이터 전송 형식 - JSON 형식

  • 자바스크립트에서 사용하는 객체 형태로 데이터를 표현 (Ajax사용시 표준으로 사용됨)
    [( "name": "김혜선", "age": "37" )]
  • 객체, 배열, 문자열, 숫자, 불, null만 들어갈 수 있음 (문자열은 큰따옴표 사용)
  • 가독성이 좋고 적은 용량으로 데이터를 전달할 수 있어 가장 많이 사용됨
  • 데이터의 양이 커지면 분석 속도가 떨어짐
